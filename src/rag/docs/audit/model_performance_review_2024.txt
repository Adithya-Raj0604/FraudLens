Issued By: Internal Audit, Model Risk Practice

SCOPE
This review assessed the ongoing performance of the XGBoost fraud detection model in production, focusing on: (1) classification performance against confirmed fraud outcomes, (2) threshold appropriateness, (3) feature stability, and (4) compliance with OSFI Guideline E-23 model risk management requirements.

MODEL PERFORMANCE SUMMARY (Q3–Q4 2024)
Total transactions scored: 4,218,433
Transactions flagged (risk_score >= 0.9828): 12,447 (0.295% alert rate)
Confirmed fraud cases in scoring population: 1,089
True positives (fraud flagged by model): 979 (recall = 89.90%)
False positives (non-fraud flagged by model): 11,468 (precision = 7.87% of flagged)
True negatives: correctly cleared: 4,205,896
False negatives (fraud missed): 110

Model AUC-ROC: 0.9984 (above minimum threshold of 0.98 — no action required)
Model PR-AUC: 0.9709

THRESHOLD ANALYSIS
The current threshold of 0.9828 was validated as appropriate for the current operating environment. Lowering the threshold to 0.97 would increase recall to 93.5% but increase alert volume by 40%, creating analyst capacity concerns. Raising the threshold to 0.99 would reduce alert volume by 22% but drop recall to 84.1%, below the 85% minimum. The current threshold is maintained.

RECOMMENDATION: velocity_24hr PSI of 0.18 should be monitored monthly. If it exceeds 0.25 in Q1 2025, a model retraining must be initiated.
